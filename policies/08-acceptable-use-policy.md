# Politica de Uso Aceptable

**Fullsite Technologies S.A. de C.V.**
Version: 1.0
Fecha de vigencia: 2026-05-25
Proxima revision: 2026-11-25
Aprobado por: Daniel Ramonfaur, CEO

---

## 1. Proposito

Definir el uso aceptable de los sistemas, dispositivos y recursos de Fullsite por parte de empleados y contratistas.

## 2. Uso aceptable

### Sistemas y herramientas
- Usar sistemas de Fullsite unicamente para propositos de trabajo
- Mantener credenciales seguras y no compartirlas
- Reportar inmediatamente cualquier sospecha de compromiso de cuenta
- Mantener software y sistemas operativos actualizados

### Codigo y repositorios
- No commitear secrets, tokens, passwords o API keys en repositorios
- No incluir datos reales de clientes en codigo, tests o documentacion
- Seguir el proceso de code review antes de merge a produccion
- Documentar cambios significativos en PRs

### Datos de clientes
- Acceder solo a datos necesarios para la tarea actual
- No copiar datos de clientes a dispositivos personales
- No compartir datos de clientes con terceros no autorizados
- No usar datos de produccion en entornos de desarrollo

## 3. Uso prohibido

- Acceder a datos de clientes sin necesidad de negocio
- Compartir credenciales o accesos con personas no autorizadas
- Desactivar controles de seguridad (MFA, encriptacion, etc.)
- Instalar software no autorizado en sistemas de produccion
- Usar sistemas de Fullsite para actividades ilegales
- Almacenar datos de clientes en servicios no aprobados (Google Drive personal, USB, etc.)
- Responder a datos de clientes en canales publicos

## 4. Dispositivos

### Dispositivos de trabajo
- Encriptacion de disco completo habilitada
- Password/biometrico en pantalla de bloqueo
- Software antivirus/antimalware actualizado (si aplica al OS)
- Actualizaciones de sistema operativo aplicadas dentro de 1 semana

### Dispositivos personales (BYOD)
- No se permite acceder a datos de clientes desde dispositivos personales
- Acceso a herramientas de comunicacion (Slack, Telegram) permitido
- Acceso a codigo fuente solo desde dispositivos con encriptacion de disco

## 5. Comunicaciones

- No enviar datos sensibles (passwords, tokens, datos de clientes) por email, Slack o Telegram en texto plano
- Usar canales encriptados para compartir informacion confidencial
- No publicar informacion interna en redes sociales

## 6. Consecuencias

El incumplimiento puede resultar en:
- Advertencia formal
- Revocacion temporal de accesos
- Terminacion de contrato
- Acciones legales si aplica
